Create `backend/app/schemas/dashboard.py` with the exact Pydantic models from RESEARCH.md "Pydantic DashboardOut Schema":
- `ColumnTimeOut(column: str, avg_hours: float)`
- `WorkloadItemOut(user_id: str, user_name: str, total_hours: float)`
- `DeptBreakdownItemOut(department_id: str, department_name: str, ticket_count: int, avg_cycle_hours: Optional[float])`
- `ThroughputPointOut(week: str, count: int)`
- `DashboardOut` with all fields: `open_ticket_count: int`, `overdue_count: int`, `throughput_last_week: int`, `avg_cycle_time_hours: Optional[float]`, `column_times: list[ColumnTimeOut]`, `workload: list[WorkloadItemOut]`, `dept_breakdown: list[DeptBreakdownItemOut]`, `throughput_trend: list[ThroughputPointOut]`

Create `backend/app/routers/dashboard.py` with `router = APIRouter()` and a single `GET /dashboard` endpoint (auth required — `Depends(get_current_user)`):

1. **open_ticket_count**: `COUNT(Ticket.id) WHERE status_column != 'Done'`
2. **overdue_count**: `COUNT WHERE due_date < date.today() AND status_column != 'Done'`
3. **throughput_last_week**: `COUNT(ColumnHistory.ticket_id) WHERE column='Done' AND entered_at >= 7 days ago`
4. **avg_cycle_time_hours**: JOIN column_history where column='Done' to tickets in Done status; `AVG(EXTRACT(epoch, ch.entered_at - Ticket.created_at)) / 3600`. Use `aliased(ColumnHistory)` as `done_ch`. Follow RESEARCH.md Pattern 3 and Pitfall 3 exactly.
5. **column_times**: GROUP BY ColumnHistory.column WHERE exited_at IS NOT NULL; `AVG(EXTRACT(epoch, exited_at - entered_at)) / 3600`. Follow RESEARCH.md "Avg Time Per Column" code example exactly.
6. **throughput_trend**: last 8 weeks bucketed by `date_trunc('week', entered_at)` WHERE column='Done'. Follow RESEARCH.md "Throughput Trend" code example. Format week as ISO string.
7. **workload**: GROUP BY Ticket.owner_id WHERE owner_id IS NOT NULL AND effort_estimate IS NOT NULL AND status_column NOT IN (Backlog, Done). Then do a second query to get `User.full_name` for each owner_id in the result (or use `selectinload` if the Ticket model has owner relationship). Build `WorkloadItemOut` list. (SQL NULLs are naturally excluded from SUM — per RESEARCH.md Open Question 1, exclude NULL effort tickets from workload.)
8. **dept_breakdown**: GROUP BY Ticket.department_id with outerjoin to ColumnHistory for avg cycle time. JOIN Department to get `department_name`. Follow RESEARCH.md "Department Breakdown Query" code example.

Register the router in `backend/app/main.py`: `app.include_router(dashboard_router, prefix="/api", tags=["dashboard"])`.

Use `StatusColumn` enum values from `app.models.ticket` for all column comparisons. Use `Depends(get_db)` and `Depends(get_current_user)` same as board router.
